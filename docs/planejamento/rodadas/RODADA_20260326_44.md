# RODADA 20260326 44

## Objetivo

Sanear `docs/projeto/manual_gestao.md`, eliminando conflitos com o bloco documental ja saneado e reposicionando o arquivo como manual operacional resumido de gestao do projeto.

## Contexto

O documento legado acumulava um regime completo concorrente ao estado atual da documentacao, incluindo:

- issue tratada como fase;
- milestones temporais;
- branches `type/*` como padrao central;
- taxonomia antiga de labels;
- blocos extensos de Git flow, review, emergency, releases e flows futuros.

A reanalise ampliada mostrou que esses temas ja haviam sido redistribuidos e saneados em artefatos proprios de `docs/flows/` e `docs/projeto/`.

## Intervencao

- reescrita integral de `docs/projeto/manual_gestao.md`;
- preservacao apenas do nucleo legitimo de visao operacional integrada;
- remissao dos detalhes normativos e tecnicos para os documentos especializados;
- adicao de nota historica curta registrando as mudancas de regime aplicadas na reescrita.

## Resultado

`manual_gestao.md` deixou de funcionar como governo paralelo antigo do projeto e passou a servir como mapa operacional resumido para:

- issues;
- milestones;
- GitHub Project;
- relacao entre issue, branch e pull request;
- consulta dos documentos corretos quando o assunto exigir maior profundidade.

## Observacoes

- esta rodada nao alterou `mkdocs.yml`;
- os conflitos foram resolvidos no proprio corpo do manual, sem manter regras concorrentes ativas;
- a parte historica foi mantida apenas como registro das mudancas de regime aplicadas ao documento.
