# Rodada — Especificações Textuais de Casos de Uso Prioritários

## 1. Contexto da rodada

Esta rodada registra a abertura da camada de **especificações textuais** da frente de casos de uso.

Ela sucede diretamente:

- o catálogo inicial de casos de uso
- os quatro diagramas de casos de uso
- a revisão integrada do conjunto já produzido

## 2. Objetivo da rodada

O objetivo desta rodada foi detalhar, em linguagem estruturada, os casos de uso considerados mais maduros e mais centrais para a continuidade da frente.

Foram escolhidos casos capazes de cobrir:

- sistema atual
- transição arquitetural
- sistema-alvo

## 3. Casos de uso priorizados

Foram especificados os seguintes casos:

- `ObterDocumento`
- `TraduzirDocumento`
- `AnalisarDocumento`
- `ConfigurarPipeline`
- `ExecutarPipelineDocumental`
- `RevisarTraducao`

## 4. Decisões tomadas

Foram tomadas as seguintes decisões nesta rodada:

- usar uma estrutura textual inspirada em práticas clássicas de especificação de casos de uso, mas adaptada ao estágio atual da frente
- preservar a distinção entre sistema atual e sistema-alvo dentro das próprias especificações
- registrar explicitamente ambiguidades e dívidas técnicas ainda abertas, sem forçar resolução prematura
- tratar os casos escolhidos como base da futura ponte com a arquitetura

## 5. Arquivos afetados

Foram criados os seguintes arquivos:

- `docs/modelagem/13_especificacoes_textuais_de_casos_de_uso_prioritarios.md`

## 6. Resultado produzido

Como resultado, a frente passa a contar com um primeiro conjunto formal de especificações textuais, aprofundando os casos mais relevantes já modelados graficamente.

## 7. Próximo passo recomendado

O próximo passo recomendado é escolher entre:

- abrir especificações textuais dos casos de uso secundários
- ou avançar para as matrizes de rastreabilidade entre atores, requisitos e casos de uso
