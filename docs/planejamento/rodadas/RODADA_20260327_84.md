# RODADA 20260327 84

## Objetivo

Executar um subciclo curto de polimento de navegacao e preparacao do site
documental para publicacao via `gh-pages`.

## Contexto

Depois do saneamento semantico da arvore `docs/`, o site documental ja estava
estruturalmente coerente, mas ainda havia dois tipos de ajuste util:

- ampliacao moderada do `nav` para melhorar descoberta humana de blocos
  relevantes
- reducao de links absolutos de filesystem em documentos vivos e publicos, que
  prejudicavam a portabilidade do site no GitHub Pages

## Intervencoes executadas

- ampliacao controlada do `nav` em `mkdocs.yml`, com inclusao de ancoras para
  `Modelagem` e para o bloco principal de `Planejamento`
- normalizacao de links absolutos nos documentos publicos, no nucleo vivo de
  `docs/projeto/`, em `docs/guias/`, `docs/metricas/` e em historicos mais
  diretamente expostos na navegacao
- substituicao de referencias quebradas a artefatos de branch de contexto por
  mencoes textuais historicas em
  `reanalise_etapa_4_publicacao_e_arquitetura_documental.md`
- ajuste do `README.md` para refletir o caminho atual de `docs/historico/fases/`

## Validacao

Foi executado:

```bash
poetry run mkdocs build --strict
```

O build concluiu com sucesso apos os ajustes deste subciclo.

## Resultado

O site documental ficou mais apto para publicacao publica:

- melhor navegabilidade para leitura humana
- menos dependencia de links locais de filesystem
- eliminacao dos warnings que ainda impediam a validacao estrita nesta etapa
