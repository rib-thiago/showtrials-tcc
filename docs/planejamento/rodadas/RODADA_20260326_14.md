# Rodada — Diagrama de Sequência dos Fluxos Atuais

## 1. Contexto da rodada

Esta rodada registra o quinto artefato do bloco UML complementar: o diagrama de sequência dos fluxos atuais.

## 2. Objetivo da rodada

O objetivo desta rodada foi consolidar quais fluxos atuais devem ser priorizados num diagrama de sequência.

## 3. Decisões tomadas

- priorizar `ObterDocumento`, `TraduzirDocumento` e `AnalisarDocumento`
- manter leitura predominantemente síncrona dos fluxos atuais

## 4. Inferências adotadas

- CLI e Web podem ser tratadas no mesmo recorte de sequência, salvo necessidade posterior de maior detalhamento

## 5. Suposições operacionais

- o diagrama de sequência atual servirá como ponte entre use cases e código real já existente

## 6. Dívidas técnicas abertas

- necessidade futura de separar sequência por interface
