# Rodada 34 - Saneamento profundo do protocolo de qualidade

## Data

2026-03-26

## Contexto

Esta rodada deu continuidade a Etapa 5 da frente de saneamento documental, agora no documento `docs/flows/quality_flow.md`.

A analise foi conduzida em contexto ja melhor estabilizado por:

- politica de governanca saneada
- protocolo de Git saneado
- documento proprio para automacao com `taskipy`
- verificacao do CI real, hooks locais e `pyproject.toml`

## Objetivo da rodada

- sanear profundamente o documento de qualidade
- separar o protocolo vigente do endurecimento historico de epoca
- renomear o arquivo fisico para refletir seu estatuto semantico
- registrar como divida tecnica os comandos de `taskipy` que o documento legado mencionava sem lastro atual

## Estado encontrado

- o documento possuia nucleo util como protocolo normativo
- havia excesso de exemplos, checklists, metadados e afirmacoes fortemente situadas no endurecimento da milestone da epoca
- parte dos comandos mencionados nao estava confirmada no `pyproject.toml` atual
- os nomes de referencia para governanca e Git ja estavam desatualizados

## Decisoes tomadas

- renomear `quality_flow.md` para `protocolo_de_qualidade.md`
- manter no protocolo apenas o nucleo semantico de qualidade
- deslocar o contexto de endurecimento para um documento historico de fase
- registrar no documento de `taskipy` as dividas tecnicas relativas aos comandos nao confirmados
