# RODADA_20260325_37

## Objetivo da rodada

Integrar os diagramas renderizados em `SVG` ao repositório de forma coerente com a política de diagramas versionáveis, melhorando a leitura dos diagramas no GitHub sem perder a primazia dos arquivos-fonte.

## Base de evidência consultada

- `docs/projeto/politica_de_diagramas_versionaveis.md`
- estrutura atual de `docs/modelagem/diagramas/`
- documentos dos diagramas de casos de uso `08-11`
- documentos dos diagramas materializados `32`, `34` e `38`
- `SVGs` gerados externamente a partir dos arquivos `PlantUML`

## Decisoes tomadas

- criar e adotar `docs/modelagem/diagramas/renderizados/` como diretorio para artefatos derivados de leitura humana
- integrar ao repositório os `SVGs` dos quatro diagramas de casos de uso e dos tres diagramas UML recentemente materializados
- manter os arquivos `.puml` como fonte principal e os `SVGs` como artefatos derivados para consulta no GitHub
- remover os arquivos `Zone.Identifier`, por serem residuos do sistema operacional e nao artefatos validos do projeto
- ajustar os documentos correspondentes e o indice geral para apontarem tambem para as visualizacoes renderizadas

## Inferencias adotadas

- a utilidade pratica dos diagramas no GitHub aumenta significativamente quando ha um renderizado navegavel no repositório
- a separacao entre `fontes/`, `especificacoes/` e `renderizados/` melhora a governanca dos diagramas sem confundir papeis

## Suposicoes operacionais

- novos diagramas relevantes poderao, quando fizer sentido, ganhar tambem artefatos derivados em `SVG`
- a presenca dos `SVGs` nao altera o estatuto semantico dos documentos de especificacao nem substitui a necessidade de manter as fontes versionadas

## Dividas tecnicas e documentais abertas

- avaliar futuramente se o projeto precisa padronizar tambem o processo de exportacao dos `SVGs`
- decidir se outros diagramas ainda preparatorios do bloco UML complementar devem ganhar renderizados no futuro, apos materializacao das fontes
