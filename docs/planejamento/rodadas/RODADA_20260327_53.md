# RODADA_20260327_53

## Objetivo

Sanear os documentos publicos de entrada ligados ao fluxo de contribuicao e registrar, com base em execucao real, o estado atual de testes, cobertura e do workaround NLP do projeto.

## Artefatos Trabalhados

- `README.md`
- `docs/contributing.md`
- `docs/projeto/dependencias_nlp_estado_e_transicao.md`

## Contexto

Durante a preparacao do saneamento do `README`, surgiram inconsistencias entre a documentacao publica e o estado real do projeto:

- comandos antigos de execucao e testes
- badge de cobertura congelado em valor antigo
- clone URL placeholder
- ausencia de explicacao pratica sobre o workaround NLP fora do `poetry install`

Tambem foi necessario validar se a suite completa de testes ainda rodava de forma confiavel no estado atual do repositorio.

## Evidencias Operacionais Obtidas

- os comandos reais do projeto hoje estao concentrados no `taskipy`
- o CI continua dependendo de um workaround hibrido para o bloco NLP
- a suite completa de testes roda no branch atual quando o workaround NLP e reproduzido no ambiente Poetry
- a fase inicial de `collecting` pode parecer lenta por causa de imports pesados de NLP, visualizacao e traducao, sem caracterizar travamento por si so

## Validacao Executada

Suite validada localmente em `2026-03-27` com:

- `235` testes passando
- cobertura total de `75.51%`

Comando utilizado:

```bash
poetry run python -m pytest src/tests --cov=src --cov-report=term-missing --cov-report=xml:coverage.xml
```
