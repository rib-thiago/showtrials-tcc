# Rodada 33 - Saneamento profundo do protocolo de Git

## Data

2026-03-26

## Contexto

Esta rodada deu continuidade a Etapa 5 da frente de saneamento documental, agora no documento `docs/flows/git_flow.md`.

Diferentemente da primeira leitura inicial, esta rodada reabriu a analise usando fontes primarias adicionais, incluindo:

- politica de governanca saneada
- historico Git do documento
- grafo de commits do projeto
- estado atual de `pyproject.toml`
- workflow de CI

## Objetivo da rodada

- sanear profundamente o documento de Git
- separar protocolo de Git, automacao com `taskipy` e versionamento/releases
- renomear o arquivo fisico para refletir melhor seu estatuto semantico
- registrar explicitamente pendencias nao confirmadas como divida de validacao

## Inputs analisados

- `docs/flows/git_flow.md`
- `docs/flows/politica_de_governanca.md`
- `docs/planejamento/plano_frente_saneamento_documentacao_legacy.md`
- `docs/planejamento/padrao_minimo_documental_etapa_5.md`
- `pyproject.toml`
- `.github/workflows/ci.yml`
- `.pre-commit-config.yaml`
- `git log --graph --decorate --oneline --all`
- historico Git relacionado a `git_flow`, `quality_flow`, CI e taskipy

## Estado encontrado

- o documento possuia nucleo util como protocolo normativo
- havia mistura excessiva entre protocolo, automacao, versionamento, release e afirmacoes nao confirmadas
- `taskipy` possui lastro real no projeto
- o historico real de merges mostra uso recorrente de merge commits
- portanto, a afirmacao de squash merge como regra consolidada nao podia permanecer sem qualificacao
- a protecao da branch `main` nao foi confirmada nesta frente

## Decisoes tomadas

- renomear `git_flow.md` para `protocolo_de_git.md`
- manter no protocolo apenas o nucleo de Git, branches, commits, PRs e merge
- mover automacao com `taskipy` para artefato proprio
- mover versionamento e releases para artefato proprio
- registrar como pendencia do protocolo tudo o que nao foi confirmado em fonte primaria suficiente

## Acoes executadas

- rename de `docs/flows/git_flow.md` para `docs/flows/protocolo_de_git.md`
- saneamento profundo do protocolo principal
- criacao de `docs/flows/automacao_operacional_com_taskipy.md`
- criacao de `docs/projeto/versionamento_e_releases.md`
- criacao do documento de rodada correspondente

## Arquivos afetados

- `docs/flows/protocolo_de_git.md`

## Riscos / pendencias

- referencias de navegacao e links ainda precisarao ser revisitados em consolidacao posterior
- a estrategia de merge do projeto ainda exige discussao propria de processo
- a protecao remota da branch `main` continua pendente de validacao

## Proximo passo recomendado

Prosseguir com analise profunda de `quality_flow.md`, agora ja em contexto mais limpo e melhor separado entre governanca, Git, automacao e versionamento.

## Observacoes de governanca

- esta rodada reforcou a regra de nao se basear apenas em documentacao isolada
- o saneamento foi conduzido com reconsulta de contexto, plano da frente e fontes tecnicas do projeto
