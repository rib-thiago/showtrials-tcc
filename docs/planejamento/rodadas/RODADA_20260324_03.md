# Rodada — Consolidação de Insumos de Requisitos para Sistema Atual e Evolução

## Data

`24/03/2026`

## Contexto

Depois da criação da primeira versão integrada do Documento de Requisitos da nova frente, surgiu a necessidade de fortalecer a base analítica do trabalho com duas visões complementares:

- uma visão focada no **ShowTrials atual**
- uma visão focada na **evolução para engine**

A intenção foi registrar esses dois insumos de forma independente, para preservar a história de maturação do documento principal e manter material de consulta futura.

## Objetivo da Rodada

Produzir e persistir dois documentos de apoio:

1. um Documento de Requisitos do ShowTrials atual
2. um Documento de Requisitos da evolução para engine

## Estado Encontrado

Foi confirmado que:

- o ShowTrials atual já permite reconstrução de requisitos retrospectivos suficientemente fortes
- a evolução para engine já possui requisitos arquiteturais explícitos em issues e documentos de visão
- o Documento de Requisitos inicial da frente mistura, de forma útil, mas ainda agregada, os três planos:
  - sistema atual
  - transição
  - sistema-alvo

## Inferências e Leitura do Codex

- separar os dois insumos em documentos próprios fortalece a rastreabilidade
- esses documentos não substituem o Documento de Requisitos principal, mas passam a funcionar como base de apoio para sua próxima evolução
- a organização em `docs/modelagem/requisitos/apoio/` é mais adequada do que misturar esses textos ao lado dos artefatos principais da frente

## Decisões Tomadas

- persistir os dois insumos em documentos separados
- tratá-los como documentação de apoio
- preservar o Documento de Requisitos principal como artefato integrador da frente
- registrar esta rodada como etapa explícita da maturação documental
