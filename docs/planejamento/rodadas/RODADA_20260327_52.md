# RODADA 20260327 52

## Objetivo

Reformular `docs/changelog.md` como changelog publico de marcos notaveis do projeto e criar um guia especifico para sua manutencao futura.

## Contexto

O changelog existente usava moldura de `Keep a Changelog` e `Semantic Versioning`, mas o historico real do projeto hoje esta muito mais fortemente sustentado por:

- historico Git;
- documentos de fase;
- documentos de rodada;
- frente de modelagem;
- frente de saneamento documental legado.

Ao mesmo tempo, o repositorio local nao apresenta tags que sustentem um regime estrito de releases versionadas. Isso tornou inadequado manter o changelog como se ele registrasse um ciclo formal de versoes ja consolidado.

## Intervencao

- reescrita integral de `docs/changelog.md`;
- substituicao do modelo antigo por entradas resumidas baseadas em marcos historicos relevantes;
- preservacao de rastreabilidade por referencia a fases, rodadas e frentes importantes;
- criacao de `docs/flows/guia_de_atualizacao_do_changelog.md` para disciplinar manutencao futura do changelog;
- articulacao do novo guia com o regime documental do projeto e com o guia geral de documentacao.

## Resultado

O changelog passou a funcionar como camada publica resumida da evolucao do projeto, enquanto o novo guia explicita:

- o que deve entrar;
- o que nao deve entrar;
- quais fontes devem sustentar uma atualizacao;
- e como relacionar changelog, Git, fases e rodadas sem gerar duplicacao indevida.

## Observacoes

- esta rodada nao implementou tags ou politica formal nova de releases;
- o changelog foi reposicionado de forma mais aderente ao lastro historico real do projeto;
- esta rodada nao alterou `mkdocs.yml`.
